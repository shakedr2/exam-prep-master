

# הפיכת האפליקציה ממבחנת למלמדת — הסבר תיאורטי וטיפים לכל שאלה

## שינויים

### 1. הוספת שדות חדשים לממשקי השאלות (`src/data/questions.ts`)
- `theoryIntro: string` — הסבר תיאורטי קצר על הקונספט (למשל: "מה זה range? איך לולאת for עובדת?")
- `approachTip: string` — טיפ איך לגשת לשאלה ("קרא את הקוד שורה-שורה, עקוב אחרי ערכי המשתנים")
- הוספת שדות אלו לכל 4 ממשקי השאלות (Quiz, Tracing, Coding, FillBlank)

### 2. יצירת קומפוננטת TheoryCard (`src/components/TheoryCard.tsx`)
- מציגה את ההסבר התיאורטי בכרטיס מעוצב עם אייקון ספר
- מציגה את הטיפ לגישה עם אייקון מפתח
- כפתור "הבנתי, בוא נתחיל" שמעביר לשלב הבא

### 3. שינוי זרימת התרגול (`src/pages/TopicPractice.tsx`)
- הזרימה החדשה: **תיאוריה → חימום → שאלה** (במקום חימום → שאלה)
- state חדש `theoryDone` שמנהל את המעבר
- אם אין theoryIntro לשאלה — מדלג ישירות לחימום/שאלה

### 4. מילוי תוכן לכל השאלות (`src/data/questions.ts`)
- הוספת `theoryIntro` ו-`approachTip` לכל שאלה קיימת
- הוספת `warmupQuestions` לשאלות שחסרות (כמו t10)
- הרחבת שאלות חימום קיימות — מ-1-2 ל-2-3 שאלות לפחות

### 5. פרטים טכניים

| קובץ | שינוי |
|------|-------|
| `src/data/questions.ts` | הוספת שדות `theoryIntro`, `approachTip` לממשקים + מילוי לכל השאלות + warmups חסרים |
| `src/components/TheoryCard.tsx` | קומפוננטה חדשה — כרטיס תיאוריה + טיפ |
| `src/pages/TopicPractice.tsx` | זרימה תלת-שלבית: תיאוריה → חימום → שאלה |
