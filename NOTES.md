# Notes on the `.lovable` Directory

## What is it?

The `.lovable` directory was generated by [Lovable](https://lovable.dev/), an AI-powered app builder that scaffolded this project. It currently contains a single file:

- **`plan.md`** — a planning/design document (written in Hebrew) that describes a proposed feature: adding a "Concepts" practice section with a logical learning order. It outlines the new pages, components, and code changes that were planned at the time the project was scaffolded.

## Should it be kept, ignored, or deleted?

| Option | Recommendation |
|---|---|
| **Keep as-is** | Fine for now — it contains useful context about planned features and causes no harm. |
| **Add to `.gitignore`** | Recommended if the team no longer uses the Lovable platform and does not want Lovable-generated metadata tracked in version control. |
| **Delete** | Safe to delete once the contents have been reviewed and any relevant information has been captured elsewhere (e.g., as GitHub issues or a project wiki). |

## Recommendation

Add `.lovable` to `.gitignore` (or delete it) once the team has confirmed it is no longer needed. The `plan.md` content may be useful to convert into a GitHub Issue before removing the folder, so that the planned feature is not lost.

> **Do not delete the folder yet** — review `plan.md` first and migrate any relevant feature plans to GitHub Issues.
