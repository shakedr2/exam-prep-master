# Execution Rules

This document defines how work flows through this repository across humans, AI agents, and PRs.

## Core Principles
1. **No broad refactors** — every PR must be scoped to a single issue/task.
2. **No unrelated cleanup** — don't mix concerns in a single PR.
3. **Preserve working flows** — if it works in production, don't touch it unless the issue requires it.
4. **Small, reviewable PRs** — target < 400 lines of diff.
5. **Rollback path required** — every PR must describe how to revert.

## PR Lifecycle
1. Issue is opened with the appropriate template.
2. Issue is triaged (severity, phase, owner).
3. Branch is created from `main` with naming: `fix/<issue-number>-<slug>` or `feat/<issue-number>-<slug>`.
4. PR is opened referencing the issue.
5. PR template checklist is completed.
6. Agent work is disclosed in the PR.
7. At least one human review is required before merge.
8. All CI checks must pass.
9. Branch is deleted after merge.

## Merge Policy
- **main is protected**: no direct pushes.
- Squash merges preferred for feature PRs.
- Merge commits allowed for release/phase PRs.
- Force pushes are disabled.

## Agent Merge Policy
- Code generated by Claude Code, Copilot, or Codex must be reviewed by a human before merge.
- Agent-generated PRs must pass all lint, test, and build checks.
- No agent may auto-merge without explicit human approval.
- Agent work must be disclosed in the PR checklist.

## Conflict / Duplication Resolution
- If two issues or PRs overlap, flag with `duplicate` label and link in the comments.
- Claude Code is responsible for deduplication triage.
- Do not implement the same feature in two separate branches simultaneously.
